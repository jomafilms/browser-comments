## RELEASE-NOTES (fork owners)
- **Admin moved to `/admin` behind a real owner login.** First visit shows a
  "create owner account" form (single owner; signup then locks). Old
  `/?admin=SECRET` is **deprecated** but still works: `ADMIN_SECRET` is accepted
  as `Authorization: Bearer` (and legacy `?admin=`) so existing installs/scripts
  don't break. `/` now redirects to `/admin` (landing lane replaces it later).
- **New dependency:** `better-auth@^1.6.23` (email+password, sessions in Postgres,
  self-hostable — no external auth service). Reuses the existing pg pool.
- **New env vars:** `BETTER_AUTH_SECRET` (>=32 chars, `openssl rand -base64 32`)
  and `BETTER_AUTH_URL` (app base URL). See `.env.example`. Never commit real values.
- **Better Auth tables** (`user`/`session`/`account`/`verification`) are created
  by `npm run init-db` (and the lazy fallback) — no separate migrate step. DDL
  lives in `lib/db/schema-auth.ts` (generated by `@better-auth/cli generate`,
  made idempotent). Re-run generate after upgrading better-auth / adding plugins.
- **Operator branding is now editable + displayed.** Admin has an instance
  branding form + per-client and per-project overrides; client-facing `/c/{token}`
  headers show the resolved logo, company name, and a Support email link.
- **Password reset: not built** (single self-hosted owner). Recovery today =
  reset directly in the DB / re-bootstrap. Follow-up for the **email lane**
  (Resend makes reset emails trivial). 2FA: not built (tier-2 hardening note).
